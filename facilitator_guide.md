# Facilitator Guide

Prep notes and delivery tips for whoever runs this session. Keep this file separate from `lesson.md` so learners aren't distracted by teaching mechanics. Read it once before class.

---

## Before Class

- **Check the GitHub UI.** GitHub redesigns its interface often. Click through the whole exercise once and confirm the button names/locations in `lesson.md` still match — "+ icon", green **Code** button, pencil (edit) icon, **Settings → Collaborators**.
- **Have a network failure plan.** This session is 100% dependent on Wi-Fi and GitHub being up, and classroom Wi-Fi with 30 people cloning at once *will* struggle. If the network dies: switch to concepts + the merge-conflict demo (Exercise B steps work fully offline in a local repo), and run the push/pull portions when it recovers.

---

## Module 1 — Opening

Open with a 2-minute recap of the pre-class video ([Git & GitHub Zero to Hero](https://youtu.be/1I79WAZ4uSU)) and a quick "what stuck / what confused you?" Don't assume everyone watched it.

---

## Setup Check

Treat this as a hard gate. If any learner's setup can't be fixed in a few minutes, route them to the browser-only path (already documented for them in `lesson.md`) rather than letting them fall behind.

---

## Exercise A — Pacing

Learners move at very different speeds here. Watch for anyone silently stuck — a stalled clone or failed push is easy to hide. Circulate rather than waiting for hands.

---

## Exercise C — Pair Collaboration

- Pair a confident learner with a slower one — in a pair, one broken setup blocks *both* people.
- Collaborator invites arrive by email/notification and can lag a minute or two. Have owners send them at the very start of the exercise so acceptance isn't the bottleneck.
- Odd number in class? Make one group of three — one Owner, two Collaborators.
